# Problème d'optimisation de pièces dans un espace prédéfini (calendrier)
![](https://github.com/bil-ela/optimisation_calendrier/blob/main/banner.jpg)

## 1. À propos de ce projet

![](https://github.com/bil-ela/optimisation_calendrier/blob/main/probleme_optimisation_calendrier.png)



En plaçant toutes les pièces sur le calendrier, il ne doit rester que 3 cases vides. Les pièces bleues ne peuvent évidemment pas ce superposer et l'objectif ici est de pouvoir réaliser toutes les dates possible du calendrier (un jour, une date, un mois) et cela de la manière la plus optimisé possible.




On trouve ci-joint à ce repository :
- Un fichier .c qui propose une solution à ce problème. Choisir une date en particulier peut être fait grâce à ce programme mais il ressort par défaut des dates spécifiques (dates paires pour le mois de janvier, février ...)
